# 影视角色卡 / 多视图 / 表情 / 姿势 — 格式与参考库

> **何时 read**：写 character sheet / 八视图 / 表情包 / 姿势组图 prompt 时。
> **配套阅读**：六维镜头 prompt 框架走 `shot-language.md`；视觉风格库 + 媒介锁走 `visual-styles.md`。

本文件给**影视特化**的角色卡场景：八视图细化定义、角色卡左右分屏布局、表情图布局、姿势库、生成参数、媒介风格锁。

## 八视图角度定义（转身图）

八视图模拟角色 360° 匀速转身，适合动作 / 游戏 / 动画参考。

| 序号 | 视角名称 | 说明 |
|---|---|---|
| 1 | 正面 | 完全面向观察者 |
| 2 | 右 45° | 角色向右转 45°（观察者看到略偏右的正面）|
| 3 | 右 90° | 角色右侧面 |
| 4 | 右 135° | 角色向右转 135°（接近背面，可见右侧后背）|
| 5 | 背面 | 完全背向观察者 |
| 6 | 左 135° | 角色向左转 135°（接近背面，可见左侧后背）|
| 7 | 左 90° | 角色左侧面 |
| 8 | 左 45° | 角色向左转 45°（观察者看到略偏左的正面）|

**单图模式**：2×4 网格，或 8 格横排长图。**多图模式**：8 张独立图。

> **关键约束**：八视图要求角色身份（面 / 服装 / 发型 / 发色）在所有 8 张中**高度一致**。Prompt 中务必**重复完整外貌描述**——把一份固定的角色描述卡片**逐字粘贴**到每张视图的 prompt 里。

## 角色卡布局（左右分屏）

**角色卡 = 全身服装展示 + 三视图，合并成横版单图**。三视图按 front / side / back 三个标准视角组装。

- **左侧（约 40% 宽）**：全身照，展示完整服装、鞋子、配饰；背景与右侧一致
- **右侧（约 60% 宽）**：三视图（正面 / 侧面 / 背面）的 1×3 排列

**整体比例**：推荐 **16:9 横版**（如需竖版则改 9:16，三视图竖排在下方）

**Prompt 要点**：左侧全身照需在 prompt 中**特别注明展示服装细节**的要求；右侧三视图所有角度与左侧服装 **100% 一致**。

## 表情图布局

**单图模式**：

- ≤ 7 种：4+3 或 3+4 网格
- ≥ 8 种：4+4 或三行布局
- 每个面板为**胸像**（头部 + 肩部），纯色背景

**多图模式**：每种表情单独一张胸像图。

每个表情面板需：

- prompt 中明确描述该表情的肌肉状态（眉形 / 眼型 / 嘴角方向 / 肌肉张力）
- 角色基础外貌（脸型 / 发型 / 肤色 / 发色）必须在所有表情中保持一致

**常用表情清单**（询问用户时参考）：冷漠 / 淡然（默认态）、微笑 / 温柔、嗤之以鼻 / 轻蔑、愤怒 / 严厉、惊讶 / 困惑、开心大笑、若有所思、悲伤 / 难过、害羞 / 窘迫。

## 多姿势组图布局

| 数量 | 布局 |
|---|---|
| 4 个 | 2×2 |
| 6 个 | 2×3（2行3列）|
| 9 个 | 3×3（推荐）|

**姿势 prompt 规范**：每个面板需 prompt 明确：动作起始状态（"双手交叉于胸前，右脚微前，重心略偏右"）+ 肢体细节（手位置 / 头朝向 / 重心偏向）+ 整体气质关键词（powerful / elegant / candid / dynamic）。

## 姿势风格参考库（按人物类型）

| 人物特征 | 推荐姿势风格 | 典型动作 |
|---|---|---|
| 西装 / 职业装 / 正装 | 职场正式感 | 站姿 / 叉腰 / 手持文件 / 背手直立 / 坐姿开会 |
| 运动装 / 球衣 / 跑步装 | 运动动态 | 起跑 / 跳跃 / 拉伸 / 投球 / 运球 / 奔跑中 |
| 休闲装 / 街头服饰 | 生活随拍感 | 插兜 / 靠墙 / 坐台阶 / 低头看手机 / 回眸 |
| 汉服 / 古装 / 旗袍 | 古典仪态 | 拱手 / 持扇 / 回眸 / 提裙 / 端坐 / 对弈 |
| 制服（军警 / 医护 / 厨师等）| 职业场景感 | 敬礼 / 执行任务 / 操作工具 / 挺胸站立 |
| 礼服 / 晚礼服 | 优雅仪态 | 裙摆展示 / 侧身回头 / 持花 / 扶栏杆 |
| 校服 / 学生装 | 青春活力 | 背书包 / 教室场景 / 跑步 / 课间互动 |

**职场姿势详细动作描述**（其它类别同理推 9 个）：

```
① 自然站立，双手垂放，面向正前（基础中性姿势）
② 双臂交叉于胸，抬颌，自信权威
③ 单手持文件夹，另一手放松，重心微移
④ 迈步行走，步伐有力，外套随步伐轻摆
⑤ 坐在桌 / 椅边缘，腿交叠，手放于膝（半身）
⑥ 单肩靠墙，手臂松散交叉，随性冷静
⑦ 背身回眸 —— 背向观察者，头转向镜头
⑧ 双手背后，侧身直立，沉稳气场
⑨ 单手整理发丝 / 领口，另一手垂落，抓拍感
```

## 通用生成参数（视图 / 表情 / 姿势共用）

| 参数 | 规范 |
|---|---|
| 风格 | 与角色来源图风格保持一致（写实 / 动漫 / 3D 等），未指定则默认写实 |
| 光线 | 柔和均匀的摄影棚光（soft studio lighting）|
| 分辨率修饰 | prompt 末尾加：`8K, sharp focus, high detail` |
| 文字标签 | **不要**在图片中添加任何文字标签 |
| 角色一致性 | 每次调用都需在 prompt 重复完整外貌描述（**子 agent 无状态**）|
| 参考图传递 | 用户提供参考图 / 原图必须将文件路径传给 image agent |
| 批量策略 | 同类资源**一次性批量生成**，不要逐张交替 |
| 背景 | 纯白（如未指定）|
| 比例 | 单图模式 1:1；多图模式 2:3 或 1:1 |
| 面板分隔 | 细浅灰线 |

## 媒介风格锁（必填，避免训练先验飘）

按用户来源图判断媒介，写入每次调用：

| 媒介 | Prompt 必须显式写 | 同时排除 |
|---|---|---|
| **写实** | `Medium: hyper-realistic photography. Photorealistic, live-action.` | `NOT anime, NOT cartoon, NOT 3D render, NOT 2D illustration` |
| **动漫 / 2D** | `Medium: 2D anime illustration style. Hand-drawn anime aesthetic.` | `NOT photoreal, NOT live-action, NOT 3D render` |
| **3D / CG** | `Medium: stylized 3D render, CG animation style.` | `NOT photoreal, NOT anime, NOT 2D illustration` |

> **整组所有视图必须锁同一种媒介**——别第 1 张写实、第 2 张动漫，跨视图一致性立刻崩。完整说明见 `visual-styles.md` 媒介锁节。
