## 1. Midjourney Niji 7

### 定位
专为 **日系动漫/漫画/插画** 设计的模型，与 Spellbrush 联合开发。二次元生图首选。

### 二次元能力矩阵

| 画风 | 表现 | 备注 |
|------|------|------|
| jp_cel 赛璃璃 | ⭐⭐⭐⭐⭐ | 原生支持，效果最好 |
| jp_ghibli 吉卜力 | ⭐⭐⭐⭐⭐ | 水彩质感到位 |
| jp_90s 90年代 | ⭐⭐⭐⭐ | 需要强化 VHS/grain 关键词 |
| jp_gacha 厚涂立绘 | ⭐⭐⭐⭐⭐ | 服装细节极强 |
| jp_ln 轻小说 | ⭐⭐⭐⭐ | 清新感好 |
| jp_shinkai 新海诚 | ⭐⭐⭐⭐ | 背景光影好，但人物可能偏甜 |
| cn_xuan3d 玄幻3D | ⭐⭐ | 不擅长3D渲染质感，出来偏2D |
| cn_ink 水墨 | ⭐⭐⭐ | 能画但墨韵不够 |
| cn_wuxia 武侠 | ⭐⭐⭐ | 偏日系化 |
| kr_watercolor 韩系水彩 | ⭐⭐⭐⭐ | 朦胧感好 |
| kr_webtoon 条漫 | ⭐⭐⭐⭐ | 干净线条 |
| kr_paint 韩系厚涂 | ⭐⭐⭐⭐ | 面部精度高 |
| chibi Q版 | ⭐⭐⭐⭐⭐ | 2头身比例控制准确 |
| us_comic 欧美漫画 | ⭐⭐⭐ | 有但不如 V7 |
| il_pixel 像素 | ⭐⭐⭐ | 需要额外提示 |

### Prompt 写法

**核心策略**：标签堆叠 + 自然语言混合

```
格式：[画风标签], [角色描述], [动作/姿势], [场景], [光影/氛围], [画质词]
```

**风格锚点词（Niji 7 专用）**：
- 赛璃璃：`anime screencap, cel shading, flat color, studio anime`
- 厚涂立绘：`game splash art, detailed illustration, painted`
- 吉卜力：`studio ghibli, watercolor, hand-drawn, warm tones`
- 90年代：`1990s anime, retro, bold outlines, VHS aesthetic`
- 轻小说：`light novel illustration, soft pastel, delicate`
- Q版：`chibi, super deformed, 2-head-tall, kawaii`

**负向词（--no 参数）**：
```
--no realistic, photograph, 3d render, blurry, watermark, text
```

**关键参数**：
- `--ar` 控制比例（头像 1:1，立绘 2:3，壁纸 16:9）
- `--s` 风格化强度（低=更遵循 prompt，高=更艺术化）
  - 赛璃璃/平涂建议 `--s 50-100`
  - 厚涂/概念建议 `--s 150-300`
- `--c` 混沌值（低=稳定，高=意外创意）

### 已知问题
1. **国漫3D 风格出不来**：Niji 天然偏2D，即使写了 "3D render" 也会拉回2D
2. **中文 prompt 支持弱**：建议全英文
3. **面部偏模板化**：大量美少女脸谱化，需要具体描述面部差异
4. **背景可能过于精致**：Q版/简约风格时背景容易抢戏，需要明确 `simple background`

### 最佳使用场景
✅ 日系全品类首选（赛璃璃/吉卜力/厚涂/轻小说/Q版）
✅ 角色立绘和头像
✅ 风格对比测试的"标准答案"
❌ 国漫3D、写实向

## 2. Midjourney V7

### 定位
通用旗舰模型，擅长**半写实、概念艺术、摄影级渲染**。二次元非主打但能力不弱。

### 与 Niji 7 的区别

| 对比项 | V7 | Niji 7 |
|-------|-----|--------|
| 默认审美 | 偏写实/西方 | 偏日系/动漫 |
| 线条风格 | 自然/隐线条 | 清晰描边 |
| 色彩倾向 | 自然色/影视色 | 高饱和/动漫色 |
| 人物比例 | 写实比例 | 动漫比例（大眼/小鼻） |
| 最强画质 | 影视/3D CG | 2D 插画 |

### 二次元使用策略

**适用场景**：
- 半写实 → 动漫过渡风格
- 概念设计/机甲/科幻
- 场景/背景为主的图
- 欧美漫画风格

**Prompt 策略**：
- 需要主动加 `anime style, 2D, illustration` 才会走二次元方向
- 如果要纯日系 → 直接用 Niji 7 更好
- V7 的优势在于**背景精度和光影层次**超过 Niji

### 最佳使用场景
✅ 新海诚风（背景超强）
✅ 半写实/赛博朋克/科幻
✅ 写实→赛璃璃的渐变过渡
✅ 欧美漫画
❌ 纯日系平涂（用 Niji 更好）

## 3. Seedream 系列（4.5 / 5.0 / Lite）

### 定位
字节跳动的图像生成模型，集成于即梦/豆包。**中文 prompt 理解力强**，审美偏精致。

### 版本差异

| 版本 | 特点 |
|------|------|
| 4.5 | 基础版，速度快，质量稳定 |
| 5.0 | 最新版，细节更好，prompt 遵循更强 |
| Lite | 轻量版，速度最快，适合批量 |

### 二次元能力矩阵

| 画风 | 表现 | 备注 |
|------|------|------|
| jp_cel 赛璃璃 | ⭐⭐⭐ | 能画但可能偏"AI味"，线条不够干净 |
| jp_ghibli 吉卜力 | ⭐⭐⭐⭐ | 水彩感不错 |
| jp_gacha 厚涂立绘 | ⭐⭐⭐⭐⭐ | 强项，服装和光效细节好 |
| cn_xuan3d 玄幻3D | ⭐⭐⭐⭐ | 中式审美好，3D质感能出来 |
| cn_ink 水墨 | ⭐⭐⭐⭐ | 中文 prompt "水墨风" 效果不错 |
| cn_wuxia 武侠 | ⭐⭐⭐⭐ | 理解"武侠"语义 |
| kr_watercolor 韩系水彩 | ⭐⭐⭐⭐ | 精致面部和低饱和处理好 |
| kr_paint 韩系厚涂 | ⭐⭐⭐⭐⭐ | 面部精度可能是全模型最高 |
| chibi Q版 | ⭐⭐⭐ | 比例控制不稳定 |
| il_pixel 像素 | ⭐⭐ | 不擅长 |

### Prompt 写法

**核心优势：中文 prompt 效果好**

```
中文写法（推荐国漫/武侠/水墨）：
"一个身穿青色道袍的白发老者，仙气飘飘，站在云海之上，国风仙侠风格，水墨画质感，大气留白"

英文写法（推荐日韩风格）：
"anime girl with blue twin tails, gothic lolita dress, holding black umbrella, cel shading, clean linework, anime screencap style"
```

**关键发现**：
- Seedream 的中文语义理解优于其他模型，**国漫/武侠/水墨风格建议全中文 prompt**
- 英文 prompt 下表现和通用模型差异不大
- `高品质, 精细, 8K` 等中文画质词有效
- 5.0 版本支持更长的 prompt，可以堆叠更多细节

### 已知问题
1. **赛璃璃平涂不够"干净"**：倾向于加渐变和柔和处理，失去平涂的硬朗感
2. **Q版比例不稳定**：头身比控制不如 Niji
3. **可能添加额外细节**：即使 prompt 简洁，模型也会自行添加装饰元素
4. **90年代复古风不理想**：VHS/胶片颗粒感模拟弱

### 最佳使用场景
✅ 国漫系列首选（玄幻3D / 武侠 / 水墨）— 中文 prompt
✅ 韩系厚涂/精致面部
✅ 需要中文语义理解的场景
❌ 纯平涂赛璃璃（用 Niji）
❌ 像素风/复古风

## 4. 香蕉系列（Pro / 2）

### 定位
平台集成的生图模型，**指令遵循能力强，风格转换稳定**。

### 二次元能力矩阵

| 画风 | 表现 | 备注 |
|------|------|------|
| jp_cel 赛璃璃 | ⭐⭐⭐ | 能画但可能偏柔 |
| jp_ghibli 吉卜力 | ⭐⭐⭐⭐ | 色调把控不错 |
| jp_gacha 厚涂立绘 | ⭐⭐⭐⭐ | 细节丰富 |
| cn_xuan3d 玄幻3D | ⭐⭐⭐ | 3D 感可能不够强 |
| cn_ink 水墨 | ⭐⭐⭐ | 基本能画 |
| kr_watercolor 韩系水彩 | ⭐⭐⭐⭐ | 朦胧感好 |
| kr_paint 韩系厚涂 | ⭐⭐⭐⭐ | 面部精度高 |
| chibi Q版 | ⭐⭐⭐ | 比例控制中等 |

### Prompt 写法

**核心优势：指令遵循强**

```
可以用更结构化的 prompt：
"[Style: Studio Ghibli watercolor]
[Subject: A young girl running through a green meadow]
[Details: warm sunlight, wildflowers, hand-drawn animation feel]
[Mood: joyful, nostalgic]
[Quality: masterpiece, best quality]"
```

**关键特性**：
- 支持中英文，英文效果略好
- 对 JSON/结构化 prompt 有较好的解析能力
- 风格转换（给参考图 → 改风格）效果稳定
- 指令细节遵循度高——说了"不要什么"通常能听进去

### 版本差异

| 版本 | 特点 |
|------|------|
| Pro | 质量更高，速度稍慢，细节更丰富 |
| 2 | 速度更快，适合批量测试和快速迭代 |

### 已知问题
1. **风格化可能不够极端**：偏向"安全"的中间态，不像 Niji 那样风格鲜明
2. **赛璃璃的硬朗感不足**：倾向于加柔化处理
3. **极端指令（如无面部）执行力一般**

### 最佳使用场景
✅ 风格转换任务（参考图 → 新风格）
✅ 需要精确指令遵循的复杂角色描述
✅ 快速迭代和批量测试（v2）
✅ 中文 prompt 需求
❌ 需要极强风格化的场景（用 Niji）

## 5. Kling 系列（v2 / v2.1）

### 定位
快手的 AI 模型，**以视频生成为主**，图像生成为辅。偏写实/3D 渲染审美。

### 二次元能力矩阵

| 画风 | 表现 | 备注 |
|------|------|------|
| jp_cel 赛璃璃 | ⭐⭐ | 不擅长纯2D平涂 |
| jp_ghibli 吉卜力 | ⭐⭐⭐ | 能画但少了手绘温度 |
| jp_gacha 厚涂立绘 | ⭐⭐⭐ | 中规中矩 |
| cn_xuan3d 玄幻3D | ⭐⭐⭐⭐⭐ | 强项！3D渲染质感最好 |
| cn_ink 水墨 | ⭐⭐⭐ | 墨韵中等 |
| cn_wuxia 武侠 | ⭐⭐⭐⭐ | 暗沉氛围感好 |
| kr_watercolor 韩系水彩 | ⭐⭐ | 不太适合 |
| chibi Q版 | ⭐⭐ | 比例控制差 |

### Prompt 写法

**核心策略：写实/3D 方向的 prompt 效果最好**

```
国漫3D（Kling 最佳场景）：
"Chinese fantasy 3D CGI, male warrior in white silk robes, holding jade sword, 
standing above sea of clouds, Unreal Engine 5, volumetric lighting, 
flowing hair and fabric, spiritual energy effects, cinematic"
```

**关键发现**：
- 3D/CG/写实类 prompt 效果远好于2D类
- 支持中文，但英文出图质量更稳定
- 动态姿势和场景构图能力不错（视频模型优势）
- 光影渲染能力强

### 已知问题
1. **2D 平涂基本做不到**：模型天然偏 3D 渲染
2. **Q版/简约风格失败率高**：总会加入过多写实细节
3. **韩系柔和风不适合**：色彩倾向偏硬偏冷
4. **像素风无法执行**

### 最佳使用场景
✅ 国漫玄幻3D（唯一首选）
✅ 武侠/仙侠半写实
✅ 场景/背景渲染
✅ 动态姿势和战斗场景
❌ 日系2D全品类
❌ Q版/像素/韩系水彩

## 6. Kontext（Flux Kontext）

### 定位
Black Forest Labs 出品，**专注图像编辑和风格转换**，不是从零生图的模型。核心能力是**保持参考图内容不变，只改变风格/细节**。

### 二次元使用方式

Kontext 的用法和其他模型不同——它需要**输入一张参考图**，然后编辑/转换风格。

**适用任务**：
- ✅ 风格转换（真人 → 动漫、油画 → 条漫、赛璃璃 → 3D）
- ✅ 角色一致性（同一角色换服装/表情/场景）
- ✅ 细节修改（改发色、改服装、改背景，不改其他）
- ❌ 从零生成（不是它的设计目的）

### 风格转换能力

| 转换方向 | 表现 | 备注 |
|---------|------|------|
| 真人 → 日系动漫 | ⭐⭐⭐⭐ | 面部特征保留好 |
| 真人 → 吉卜力 | ⭐⭐⭐⭐ | 色调转换准确 |
| 2D → 3D | ⭐⭐⭐ | 质感变化有但可能不彻底 |
| 油画 → 条漫 | ⭐⭐⭐⭐ | 线条简化处理好 |
| Q版 → 正常比例 | ⭐⭐ | 结构变化太大，不太靠谱 |
| 跨文化风格转换 | ⭐⭐⭐⭐ | 日→中、日→韩 等 |

### Prompt 写法

```
风格转换模板：
"Transform this image into [target style]. 
Keep the character's [features to preserve: hair color, outfit, pose].
Change the art style to [specific style description]."

示例：
"Transform this image into Studio Ghibli watercolor style. 
Keep the character's face and outfit. 
Change to warm pastel tones, soft hand-drawn linework, rounded features."
```

### 已知问题
1. **不适合从零生成**：必须有参考图输入
2. **风格变化可能不够极端**：偏向保守的"安全转换"
3. **Q版↔正常比例转换不可靠**：涉及大比例结构变化
4. **复杂多人场景可能乱**
5. **⚠️ 画质低于文生图模型**（Benchmark 实测）：输出被压为 1024×1024，整体精致度和风格深度不如 Niji/Kling/Seedream 的文生图路径。跨风格转换时出图有"滤镜感"而非"重绘感"

### 最佳使用场景
✅ **细微编辑**：改发色、改服装颜色、改背景，不改其他
✅ 保持角色一致性换服装/表情
✅ 局部修改（加配饰、换眼镜）
⚠️ **风格转换不推荐**：Benchmark 实测证明，跨风格转换（如真人→动漫、2D→3D）应优先用**文生图路径**（read_media 提取特征 → 用目标风格的最优模型重新生成），画质远高于 Kontext。Kontext 仅在需要像素级构图保留时作为补充。
❌ 从零生成（用其他模型）
❌ 大幅结构变化（Q版↔正常）

## 7. nano_banana 系列（banana-2 / Pro / Flash）

### 定位
平台集成的高指令遵循生图模型。**最大参考图数量 max_refs=10**，单图输出，是**分镜/多帧一致性任务**的首选引擎。

### 核心能力

| 能力 | 表现 |
|------|------|
| max_refs | **10**（全模型最高，可传多张参考图） |
| 单图输出 | ✅（无四宫格，输出即可直接复用） |
| 指令遵循 | ⭐⭐⭐⭐⭐（强约束执行率高） |
| 中文渲染 | ✅ |
| 风格极致化 | ⭐⭐⭐（偏"安全中间态"，不如 Niji 极端） |

### 二次元分镜任务能力矩阵

| 画风 | 表现 | 备注 |
|------|------|------|
| jp_cel 赛璃璃 | ⭐⭐⭐ | 能画但风格化弱于 Niji |
| jp_ghibli 吉卜力 | ⭐⭐⭐⭐ | 色调把控好 |
| jp_gacha 厚涂立绘 | ⭐⭐⭐⭐ | 细节丰富 |
| kr_watercolor 韩系水彩 | ⭐⭐⭐⭐ | 朦胧感稳定 |
| kr_paint 韩系厚涂 | ⭐⭐⭐⭐ | 面部精度高 |
| chibi Q版 | ⭐⭐⭐ | 比例中等 |
| cn_xuan3d 玄幻3D | ⭐⭐⭐ | 3D感中等 |

### 最佳使用场景（任务类型维度）

✅ **分镜/多帧连续**：每一帧把前一帧作为参考图传入，锁定角色外貌和构图
✅ **构图固定只改光影/季节**：传上一帧 ref + 强指令约束，构图稳定性远高于纯文字
✅ **表情差分**：传角色图锁住面部，只改表情描述
✅ **三视图**：以正面图为 ref 生成侧面/背面
✅ **风格转换（参考图→新风格）**：接受原图 ref + 风格指令
❌ 需要极强风格化的独立单张图（用 Niji 7 / MJ V7 更鲜明）

### Prompt 写法

```
多帧锚定写法（分镜）：
[第N帧] 传入第N-1帧作为参考图，写明"same character same composition"
+ 本帧差异描述（仅改光线方向 / 仅改表情）

构图锁定写法：
"Same exact composition as reference image. Only change: [具体变量]. Keep all else identical."
```

### 版本差异

| 版本 | 特点 |
|------|------|
| banana-2 / Pro | 质量最高，细节丰富 |
| Flash / nano_banana_2 | 速度快，适合批量 |

## 任务类型 → 模型速查（优先于画风路由）

> 以下任务类型的选模型逻辑由**一致性/指令遵循/输出格式**驱动，不由风格决定。

| 任务类型 | 首选模型 | 原因 |
|---------|---------|------|
| 分镜/多帧（同角色/同场景 N≥2） | **nano_banana** | max_refs=10 可传前帧；单图输出；指令遵循强 |
| 构图固定只改光影/季节 | **nano_banana** | 传上一帧 ref 锁构图 |
| 表情差分（同角色多表情） | **nano_banana** | 传角色图锁面部 |
| 三视图 / 多角度立绘 | **nano_banana** | 传正面图锁角色 |
| 单张独立立绘/头像 | 按画风路由表 | 无跨图一致性需求，风格优先 |
| 风格转换（跨风格重建） | 目标风格首选引擎 | 文生图路径，见画风路由表 |
