# Benchmark 实测经验

> 基于 Benchmark v1（40 题 / 8 维度），总分 104.5/122.5（85.3%）。

## ✅ 已验证的最佳实践

1. **引擎选择是最大质量杠杆**：同一 prompt 在不同引擎上的风格准确度差异是量级的。必须查 `model-knowledge-base.md` 速查表为每种画风匹配最优引擎，不能一刀切用默认引擎。
2. **风格转换走文生图路径**：分析提取特征 → 用目标风格最优引擎文生图。编辑型引擎的图生图画质低一个量级（滤镜感），仅用于细微编辑。
3. **跨引擎策略拉开风格差距**：相近风格（如日系厚涂 vs 韩系厚涂）用不同引擎比同引擎内 prompt 分化效果好得多。
4. **角色还原靠特征完整描述**：5 个以上特征点时，需逐项列出。中文理解型引擎对位置关系理解最强（如"古琴在背上"）。

## ⚠️ 已知弱项与应对

| 弱项 | 场景 | 应对策略 |
|------|------|---------|
| 同引擎内风格细分差异度不足 | 赛璃璃 vs 轻小说 | 加大风格化参数差距 + 互加负向词排斥对方特征（详见 style-bank.md 差异化技巧） |
| 多角色发色/特征趋同 | 双胞胎一黑一白 | 分区描述（LEFT/RIGHT）+ 大写强调差异（详见 style-bank.md 多角色策略） |
| Q版→正常比例控不住 | 日系2D引擎生成仍偏 3-4 头身 | 降级到中文理解型引擎 + `NORMAL ADULT HUMAN PROPORTIONS, 7-head-tall, long slender legs` |
| 公司级亚风格难精确 | ufotable vs Trigger | 跨引擎生成，但差异仍有限 |
| 风格渐变过渡难控制 | 写实→Q版 4级渐变 | 每级用不同类型引擎，但跨引擎时面部一致性会下降 |
