# Mnemosyne Embedding Fix — 2026-06-23

## Situation

- Mnemosyne `memory_embeddings` Table hatte nur 127 Einträge (verwaist, referenzierte IDs die nicht in `working_memory` standen)
- 659 working_memories und 100 episodic_summaries hatten KEINE Embeddings
- `mnemosyne_stats.vectors: 0`, `vec_type: "none"`
- `mnemosyne.core.embeddings.available()` = `False`

## Root Cause

`fastembed` war nicht installiert. Die Embedding Engine (`BAAI/bge-small-en-v1.5`, 384 dim) kann ohne fastembed keine lokalen Vektoren generieren. Die 127 alten Embeddings waren von einer früheren Session mit API-basierter Embedding-Generierung.

## Fix Steps

### 1. fastembed installieren

```bash
uv pip install fastembed --python ~/.hermes/hermes-agent/venv/bin/python
```

**Hinweis:** `uv pip install` ohne `--python` flag installiert in den uv-Cache, nicht in den venv.

### 2. Embeddings generiert (Python Script)

```python
import sqlite3, json, numpy as np
from fastembed import TextEmbedding

embedder = TextEmbedding('BAAI/bge-small-en-v1.5')

class NpEncoder(json.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, (np.floating,)):
            return round(float(obj), 6)
        return super().default(obj)

db = sqlite3.connect('~/.hermes/mnemosyne/data/mnemosyne.db')
cur = db.cursor()

# Working Memory: 659 Embeddings generiert
cur.execute("""
    SELECT wm.id, wm.content 
    FROM working_memory wm
    LEFT JOIN memory_embeddings me ON wm.id = me.memory_id
    WHERE me.memory_id IS NULL AND wm.content IS NOT NULL
""")
for mem_id, content in cur.fetchall():
    vec = list(embedder.embed(content[:500]))[0]
    vec_json = json.dumps(vec.tolist(), cls=NpEncoder)
    cur.execute(
        "INSERT OR REPLACE INTO memory_embeddings (memory_id, embedding_json, model) VALUES (?, ?, ?)",
        (mem_id, vec_json, 'BAAI/bge-small-en-v1.5')
    )
db.commit()

# Episodic Memory: 100 Embeddings generiert
cur.execute("""
    SELECT em.id, em.content 
    FROM episodic_memory em
    LEFT JOIN memory_embeddings me ON em.id = me.memory_id
    WHERE me.memory_id IS NULL AND em.content IS NOT NULL
""")
# ... gleiches Muster
```

**Performance:** ~70 emb/s. 659+100 = ~12s.

### 3. Konsolidierung

```python
from hermes_tools import mnemosyne_sleep
mnemosyne_sleep(all_sessions=True)
# → 6 items konsolidiert, 1 neue episodic summary
```

## DB Schema Notes

- `working_memory` hat Spalte `content` (nicht `content_preview`)
- `episodic_memory` hat Spalte `content`
- `memory_embeddings.embedding_json` ist JSON-String mit float32→float64 konvertierten Werten
- `vec0` Extension erwartet `int8[384]` — die `memory_embeddings` Table nutzt JSON float arrays für Portabilität
- `vec_episodes` hat nur 20 Einträge (nicht vollständig befüllt) — optional für ANN-Performance
- `sqlite3` CLI braucht `enable_load_extension(True)` + expliziten Pfad für vec0
